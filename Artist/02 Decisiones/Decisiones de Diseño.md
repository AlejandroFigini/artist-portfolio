# Decisiones de Diseño

← [[MOC Portfolio]]

Registro de por qué las cosas están como están.

---

## Estética: Blueprint Cinematic

**Decisión:** UI con cuadrícula blueprint, cursor custom, esquinas de viewport.  
**Por qué:** Refuerza la identidad de artista técnica — mezcla de arte y proceso.

---

## Sin framework de JS

**Decisión:** HTML/CSS/JS puro, sin React ni Vue.  
**Por qué:** Portfolio estático no necesita reactivity. Más fácil de mantener para una sola persona.

---

## CMS propio

**Decisión:** CMS integrado en el mismo proyecto, no CMS externo.  
**Por qué:** Control total sobre la estructura del contenido y las galerías.
